# 🔑 트랙리드 운영 플레이북

**대상: 정별 (트랙리드).** 매주 "언제, 뭘, 어떤 순서로" 하는지의 운영 대본.
학회원들은 안 읽어도 되지만, 읽어도 상관없음 (투명하게 운영).

---

## 전체 로드맵 한눈에

```
Week 0  (이번 토)  온보딩 — GitHub 한 사이클 실습          @ SS-onboarding
Week 1            GitHub 기초 마무리 + Claude Code 입문    @ SS-onboarding
Week 2            Claude Code 기본 (settings/hooks/커맨드)  @ SS-Harness-private
Week 3~4          개인 하네스 구축 & 실사용 (2주)           @ SS-Harness-private
Week 5~           공동 하네스 통합 (해커톤식)               @ SS-Harness-communal
8월 말            학회원 전체 배포 & 사용 가이드 완성
```

---

## 매주 반복되는 리듬 (이것만 지키면 굴러감)

### 모임 전 (금요일까지)
- [ ] 이번 주 과제 Issue 발행 (`weekly-task` 라벨 + 해당 milestone + assignee 3명)
- [ ] 지난주 PR 중 안 닫힌 것 확인 → Discord로 리마인드
- [ ] 토요일에 보여줄 것/시연할 것 준비

### 모임 중 (토 11시, 90분 기준)
- [ ] 10분: 지난주 산출물 하이라이트 (좋았던 PR 1~2개 화면 공유)
- [ ] 60분: 이번 주 주제 진행 (강의 시청분 실습 / 페어로 막힌 것 해결)
- [ ] 15분: 다음 주 과제 Issue 같이 읽기 — 기대 산출물 눈높이 맞추기
- [ ] 5분: 회고 한 마디씩 (막힌 곳, 배운 곳)

### 모임 후 (일요일까지)
- [ ] 모임 중 나온 질문/결정사항 → Issue 코멘트나 Discord에 기록
- [ ] 올라온 PR 리뷰 (48시간 내 첫 리뷰가 원칙 — 늦으면 동력 떨어짐)

> 💡 리뷰 코멘트 팁: "이거 틀림" 대신 "이렇게 하면 ~한 장점이 있는데 어때요?"
> 첫 몇 주는 merge 기준을 낮게. 완벽한 PR보다 돌아가는 사이클이 중요.

## 주차별 상세

### Week 0 — 온보딩 (이번 토요일)

**목표:** 전원이 PR 1개 merge 성공. 그게 전부.

- 준비물: `docs/saturday-mission.md` 절차 본인이 미리 한 번 돌려보기 (리허설)
- 발행할 Issue: `[Week 0] 자기소개 PR 올리기` — assignee 3명, `good-first-task`
- 당일 아젠다: `saturday-mission.md` 상단 표 참고
- **주의:** 403 에러 (org 초대 미수락) 가장 흔함. 시작 전에 전원 초대 수락 확인
- 끝나기 전: 다음 주 과제(강의 시청) Issue 같이 읽기

### Week 1 — GitHub 기초 마무리 + Claude Code 입문

**과제 (모임 전까지):**
- 강의 1 시청: https://www.youtube.com/watch?v=vxEvo2BLM6A (핵심만)
- (선택) 강의 2: 인프런 하네스 강의 — 실습 파트까지 볼 사람은 보기
- 시중 스킬 조사해오기 (각자 1개 이상, `skill-share` Issue로 공유)

**모임에서:**
- 튜터가 만든 paper 관련 레포 clone → 실제 실행 → 후기 공유 + star 누르기
- OPT 계정 튜터에게 공유 → 스킬 push 받기
- conflict 상황 일부러 만들어서 해결 실습 (2명씩 같은 파일 수정 → rebase)

### Week 2 — Claude Code 기본

**과제:** settings.json, hooks, slash commands 각자 하나씩 만들어오기

**모임에서:**
- `SS-Harness-private` 레포 소개 — 본인 폴더 구조 설명
- 각자 만들어온 것 데모 (화면 공유 5분씩)
- 개인 하네스 2주 계획 각자 README 진행 로그에 작성

### Week 3~4 — 개인 하네스 구축 (2주)

**핵심 메시지:** "유저는 본인 1명. 본인이 실제로 매일 쓰는 게 성공 기준."

- 매주 PR 1개 이상 (작아도 됨 — notes 기록만이라도)
- 트랙리드는 각자 목표(폴더 README에 있음) 기준으로 리뷰
- Week 4 모임: 중간 데모 — 실사용 후기 중심으로

### Week 5~ — 공동 하네스 통합

- `SS-Harness-communal`의 `integration-task` Issue 템플릿으로 후보 수집
- 모임에서 해커톤식으로: 후보 투표 → 통합 담당 배분 → 그 자리에서 작업
- 충돌하는 설정은 그 자리에서 결정 (미루면 안 끝남)
- 최종 기준: **"신규 학회원이 docs만 보고 clone → 사용까지 혼자 가능한가"**

## 상황별 대응

| 상황 | 대응 |
|------|------|
| 과제 미제출자 발생 | 공개 압박 X. Discord 개인 DM으로 막힌 곳 물어보기 — 대부분 기술적으로 막혀서임 |
| PR이 너무 큼 | "다음엔 쪼개서 올려달라" 코멘트 + 이번 건 그냥 merge (첫 달은 관대하게) |
| 모임 불참자 | Issue에 모임 요약 코멘트 남기고, 과제는 동일하게 |
| 본인(트랙리드)이 바쁜 주 | 리뷰만은 사수. 모임은 페어 실습 위주로 돌려서 준비 부담 줄이기 |
